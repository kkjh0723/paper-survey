RT-1: Robotics Transformer for Real-World Control at Scale

Anthony Brohan, Noah Brown, Justice Carbajal, Yevgen Chebotar 외 (51인) · Chelsea Finn · Karol Hausman · Sergey Levine · Google · Everyday Robots

arXiv abstract PDF 프로젝트 페이지 GitHub arXiv:2212.06817
Robotics Transformer FiLM-EfficientNet TokenLearner Action Discretization Cross-Embodiment

제출: 2022년 12월 13일 (v2 2023년 8월) · 35M 파라미터 · 3Hz 실시간 제어 · 13대 로봇 17개월 · RSS 2023

한 줄 요약 (TL;DR)

13대 Everyday Robots를 17개월간 운용해 130k 데모·744개 언어 지시를 수집하고, FiLM-EfficientNet(언어 조건 이미지 인코더) → TokenLearner(81토큰→8토큰) → 8-layer Transformer로 11차원 액션을 256-bin으로 이산화해 3Hz 실시간 예측. 200+ seen 지시에서 97%, unseen 지시 76%, 방해물 83%·배경 59%. 20+스텝의 SayCan 롱-호라이즌 태스크를 두 실주방에서 67% 성공. VLA/generalist policy 시대의 스케일링 가능성을 처음 실증한 랜드마크 논문.

1배경 · 문제의식

비전·NLP·음성은 이미 대규모 사전학습 → 일반화 패러다임이 정착했지만 로봇 정책은 여전히 태스크별 별도 학습이었다. 로봇 데이터는 (1) 수집이 비싸고, (2) 이질적 하드웨어·환경으로 나뉘고, (3) 폭넓은 태스크 커버리지가 어려워, "task-agnostic 대규모 학습 → 일반화"가 검증되지 못했다. RT-1은 이 세 문제를 (a) 17개월 대규모 데이터 수집, (b) 고용량 트랜스포머, (c) 실시간 3Hz 제어에 맞춘 토큰 압축(TokenLearner)으로 정면 돌파한다.

2핵심 Contribution

  • Robotics Transformer 아키텍처 — FiLM-EfficientNet + TokenLearner + 8-layer Transformer로 이미지·언어를 48 토큰으로 압축해 35M 모델이 3Hz 실시간 제어 달성. TokenLearner + token reuse가 각각 2.4×/1.7× 가속.
  • 대규모 실세계 데이터셋 — 13 로봇, 17개월, 130k 에피소드, 744 언어 지시(8 skill 카테고리). 이후 Open X-Embodiment의 Fractal 데이터셋 원본이 됨.
  • 스케일링 & 일반화의 최초 대규모 실증 — seen 97% (Gato +32pt), unseen 76%, 방해물 83%. Kuka QT-Opt 데이터·시뮬 데이터 흡수 시 기존 실주방 성능 유지 + bin-picking 일반화 2× 향상. SayCan 결합으로 50-스텝 롱-호라이즌 계획 실행 시연.

3Method

FiLM-EfficientNet 이미지 인코더

ImageNet 사전학습 EfficientNet-B3가 6장의 300×300 이미지를 9×9×512 특징 맵으로 인코딩. 각 MBConv 뒤에 identity-초기화 FiLM 레이어를 삽입, Universal Sentence Encoder로 임베딩한 언어 지시를 이미지 특징에 early fusion으로 주입해 태스크-관련 특징이 강조되도록 학습. 이미지 토크나이저 총 16M 파라미터, 뷰당 81 토큰(9×9).

TokenLearner (실시간 제어의 핵심)

81 토큰을 element-wise attention으로 8 토큰으로 압축. 6장 이미지 × 8토큰 = 총 48 토큰만 트랜스포머로 흘려보내 트랜스포머 계산량이 극적으로 감소. 이 압축 덕에 8-layer 19M 트랜스포머가 3Hz로 돌아간다.

Action 이산화 & 출력

11-DoF 액션을 각 차원 256-bin으로 discretize해 language-model 스타일 토큰 예측으로 학습:

  • arm 7-DoF: x, y, z, roll, pitch, yaw, gripper
  • base 3-DoF: x, y, yaw
  • 제어 모드 1-DoF: {arm, base, terminate}
Cross-entropy 손실로 학습, argmax로 디코딩.

Kitchen1 67% · Kitchen2 67% 실행 성공률 (10+ 스텝 태스크). 최대 50-step 태스크 실행. RT-1이 low-level 정책일 때 SayCan 전체 파이프라인이 가장 안정.

이질 도메인 흡수 (Table 4·5)

  • 시뮬 데이터 흡수: 시뮬 물체 성공률 23% → 87% (+64pt), unseen skill-object 조합 7% → 33%. 실세계 성능 저하 없이 이득만 얻음 — 대규모 트랜스포머가 시뮬-리얼 도메인 shift를 흡수함을 보여줌.
  • Kuka QT-Opt 다중 로봇 데이터: Kuka bin-picking 일반화 22% → 39% (≈2×), Everyday Robots 원 태스크는 −2%p만 하락. 최초의 cross-embodiment positive transfer 실증.

5Ablation (주요 발견) — Table 7

  • 데이터 양: 100% 데이터 → unseen 76%, 51% → 52%, 22% → 14%. 데이터 양이 결정적.
  • 데이터 다양성: 75% 태스크·97% 데이터 유지 시 unseen 67%. 저자 결론: "데이터 다양성이 데이터 양보다 더 결정적" — 소량 다양 > 다량 좁음.
  • TokenLearner: 제거 시 3Hz 실시간 제어 불가능. 성능 손실 없이 2.4× 속도 향상.
  • ImageNet 사전학습: 제거 시 seen 태스크에도 유의미한 성능 하락.
  • 모델 크기 스케일링: 파라미터 증가 시 unseen·일반화 지속 향상 — 후속 RT-2·Octo·OpenVLA로 이어지는 스케일링 근거.
