SmolVLA: A Vision-Language-Action Model for Affordable and Efficient Robotics

Mustafa Shukor, Dana Aubakirova, Francesco Capuano, Pepijn Kooijmans 외 · Andres Marafioti · Simon Alibert · Matthieu Cord · Thomas Wolf · Remi Cadene (교신) · Hugging Face LeRobot 팀

arXiv abstract PDF GitHub (lerobot) arXiv:2506.01844
Efficient VLA SmolVLM-2 Flow Matching Async Inference Community Data

제출: 2025년 6월 2일 (v1) · 백본 SmolVLM-2 · 총 0.45B (action expert ~100M) · 학습 4 GPU · 배치 256

한 줄 요약 (TL;DR)

SmolVLM-2 기반의 0.45B 초경량 VLA — VLM의 앞쪽 절반만(N=L/2) 쓰고, action expert에 CA(cross)·SA(self) 교차 배치 + causal SA를 넣어 flow matching으로 n=50 액션 청크 생성. 커뮤니티 LeRobot 481개 데이터셋(22.9K 에피소드)만으로 사전학습해 LIBERO 평균 87.3(π0 86.0, OpenVLA 76.5), Meta-World 평균 57.3(π0 47.9). 저가 SO-100 실기 평균 78.3 vs π0 61.7, 비동기 추론으로 30% 빠르고 고정 시간 pick-and-place 19회 vs 동기 9회.

1배경 · 문제의식

기존 VLA(OpenVLA 7B, π0 3.3B, GR00T 2.2B 등)는 수십억 파라미터로 학습 비용·배포 난이도가 실제 로봇 커뮤니티(저가 SO-100 등)엔 진입 장벽이 크다. 또 학습 데이터도 대기업이 폐쇄적으로 보유한 실기 텔레옵에 크게 의존. 저자들은 단일 GPU 파인튜닝·소비자 하드웨어(심지어 CPU)에서 실행가능한 규모의 VLA를 공개 커뮤니티 데이터로만 만들 수 있음을 목표로 삼는다.

2핵심 Contribution

  • 초경량 VLA 아키텍처(0.45B) — SmolVLM-2 백본에서 앞쪽 절반 층만 사용(N=L/2)하고 시각 토큰은 프레임당 64개로 축소(pixel-shuffle). Action expert는 hidden 0.75×d로 슬림.
  • CA·SA 교차 배치 액션 전문가 — 각 블록이 CA 또는 SA. SA는 causal 마스크로 과거 액션만 참조. Flow matching으로 n=50 액션 청크 예측. Ablation에서 CA-only 79.0, SA-only 74.5 대비 85.5로 우수.
  • 커뮤니티 데이터만으로 SOTA급 성능 + 비동기 추론 — LeRobot 커뮤니티 481 데이터셋 / 22.9K 에피소드 / 10.6M 프레임(경쟁 모델보다 한 자리수 작음)으로 사전학습. RobotClient/PolicyServer 분리로 관측·행동을 디커플해 30% 빠르고 처리량 2배.

3Method

백본 SmolVLM-2 & 입력 처리

SigLIP 비전 인코더 + SmolLM2 언어 디코더의 다중 이미지·영상 최적화 VLM. 이미지엔 global-view + pixel-shuffle로 프레임당 64 토큰만 사용(계산 감소). 로봇 state는 선형층으로 단일 토큰. 언어 지시 텍스트와 함께 SmolVLM-2에 입력.

Layer Skipping (VLM 압축)

전체 L개 LLM 층 중 앞쪽 N=L/2 층의 hidden만 action expert에 넘긴다. Ablation(Table 8): 첫 16층 78.5, 32층(=L/2) 80.3(최적), 격층 스킵 75.5. → 후반 층의 언어·추론 특화 특징보다 중간층의 시각·기하 특징이 액션에 유리.

Action Expert (CA·SA 교차, Flow Matching)

액션 청크 길이 n=50. 각 블록이 CA(cross-attention, gold) 또는 SA(self-attention, light yellow)를 번갈아 배치. SA는 causal 마스크로 과거 액션만 attend. Hidden dim은 VLM의 0.75×d. Flow-matching으로 학습(회귀 대비 +5pt: 80.25 vs 75.25). 청크 크기 sweep: 1→50 성공률 50→84, 100은 오히려 74.5로 하락.

커뮤니티 데이터셋 표준화

Hugging Face 커뮤니티에 올라온 481개 LeRobot 데이터셋 / 22.9K 에피소드 / 10.6M 프레임. 자동/수동 전처리:

  • 태스크 설명 자동 생성: Qwen2.5-VL-3B-Instruct로 짧은 언어 지시 자동 생성.
  • 카메라 매핑: 각 카메라를 top/wrist/side로 표준화 매핑.

비동기 추론 (Async)

로봇 측 RobotClient가 관측을 보내면 PolicyServer가 액션 청크를 계산해 반환. 관측 유사도 기반 필터로 큐 사이즈를 조절(g=0 순차, g=0.7 균형, g=1 컴퓨트 집중). 관측 취득과 계산을 겹치게 해 실시간성을 확보.

Async vs Sync 추론

동일 태스크 완료 시간 9.7s(async) vs 13.75s(sync) — ~30% 빠름. 고정 시간 내 pick-and-place 성공 횟수 async 19회 vs sync 9회 — 처리량 2배.

5Ablation (주요 발견)

  • Attention 배치 (Table 6): CA-only 79.0 / SA-only 74.5 / Interleaved CA+SA 85.5 — 교차 배치가 결정적.
  • VLM Layer 개수 (Table 8): 첫 32층(=L/2) 80.3(최적) / 16층 78.5 / 격층 스킵 75.5 — 앞쪽 절반이 sweet spot.
  • Loss (Table 10): Flow matching 80.25 vs L1 회귀 75.25 — flow matching이 +5pt.
  • Chunk Size (Table 12): chunk 1 → 50.0, chunk 10~50 → 80~84 (sweet spot), chunk 100 → 74.5 (과다 롤아웃 예측 부담).
