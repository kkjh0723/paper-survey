한 줄 요약 (TL;DR)

코딩 에이전트(Codex·Claude Code·Kimi)가 실세계 로봇 정책을 사람 개입 없이 자율 개선하는 harness. 2단계로 나눠, (1) 사람 안내로 리셋·리워드·안전제약 환경 API를 한 번 구축하고, (2) 그 뒤 에이전트가 로그 분석 → 가설 → 학습코드 수정 → 롤아웃 → 자동 검증 루프로 성공률을 hill-climb 한다. 8대 YAM 양팔 로봇 함대에 N개 에이전트를 붙여 Git으로 학습 recipe를 공유(분산 협업)한다. Pin insertion에서 100%(50연속 성공)로 human-in-the-loop을 앞질렀고, 에이전트를 1→8개로 늘리면 수렴 시간이 1.5시간+ → ~40분으로 단축된다. "physical autoresearch"를 정식 문제로 정의하고 자원 효율 지표(MRU·MTU)를 제안.

1배경 · 문제의식

실세계 정교(dexterous) 조작은 데이터 수집·평가/리셋 사이클·알고리즘 엔지니어링에 사람 손이 크게 든다 — 물리적 지능을 스케일업하는 핵심 병목이다. 코딩 에이전트는 디지털 환경의 연구를 자동화하는 데는 성공했지만, 실세계로 옮기면 두 가지가 없다: (1) 닫힌 루프 가설 검증을 위한 실세계 환경 인터페이스(자동 리셋·배포·검증), (2) 자원 효율을 지키며 로봇 함대로 확장하는 메커니즘. 저자들은 이 문제를 physical autoresearch로 정식화한다.

2핵심 Contribution

  • physical autoresearch 정식화 — 사람이 안내하는 환경 구축과 실세계 피드백 기반의 완전 자율 정책 개선을 결합한 문제로 정의.
  • ENPIRE harness — 코딩 에이전트가 재사용 가능한 로봇 툴을 개발하고, tool call로 리워드·리셋을 구성하며, 실로봇에서 학습 알고리즘을 개입 없이 최적화.
  • 확장성 실증 — dexterous 태스크에서 99% 성공, 에이전트 1→8개로 pin insertion 수렴을 1.5시간+에서 ~40분으로 단축.
  • 효율 지표 제안 — Mean Robot Utilization(MRU), Mean Token Utilization(MTU)으로 physical autoresearch의 자원 소비를 벤치마크.

2단계 피드백 루프

Stage 1 — 환경 구축(사람 안내, 1회): 에이전트가 SAM3(세그멘테이션)·cuRobo(충돌 계획)·BundleSDF(포즈 추적) 같은 tool call로 리셋 함수를 합성하고, 성공/실패 데모로부터 이진 리워드(시각 정렬·힘 피드백·삽입 깊이)를 설계하며, 하드 안전 제약(관절 한계·접촉 임계)을 구현한다. 이 비용은 1회성으로 이후 태스크에 amortize된다.

Stage 2 — 자율 정책 개선(온라인): 에이전트는 태스크 설명과 불변 Gym API 환경만 받아, 궤적 로깅 → 통계 관찰 → 가설 수립 → 학습 코드 수정 → 롤아웃 → 자동 리워드 검증을 반복한다. heuristic 학습, behavior cloning(BC), online/offline RL, code-as-policy 등 다양한 방법을 탐색한다.

분산 함대 협업(Evolution): N개 에이전트를 N대 로봇에 붙여 N개 학습 recipe를 비동기로 시험하고, Git 브랜치를 병합·cherry-pick 해 성공 recipe를 흡수하고 실패 아이디어는 버린다. 원문: "agents spontaneously cherry-pick, copy, or merge successful training recipes".

5Ablation (주요 발견)

  • 비전 방식(Codex): 네이티브 비전이 가장 빠르고, function-call 비전은 중간(반복 호출 오버헤드), 비전 없이도 성공 — 에이전트가 텍스트 로그만으로 상태를 추론.
  • 모델·harness 비교: 단순화 Push-T에서 Codex가 가장 빠르게 해결, Claude Opus 4.7은 이 태스크에선 상대적으로 덜 효율적.
  • Idea tree(Pin Insertion): BC regularization이 +10.8pp로 진전의 대부분을 차지, 이후 batch size(1024→512)·controller 보정 등은 +0.9~1.3pp의 미세 개선.
