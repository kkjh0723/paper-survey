Reflex: Real-Time VLA Control through Streaming Inference

Yuanchun Guo, Bingyan Liu · NSFC 62302054 지원 · ICML 2026 채택

arXiv abstract PDF GitHub arXiv:2607.14695
VLA Flow Matching Streaming Inference KV Cache Real-Time

제출: 2026년 7월 16일 · 백본 π0 / π0.5 / SmolVLA · RTX 4090 (3090 검증)

한 줄 요약 (TL;DR)

Flow-matching VLA의 반복적 denoising이 실시간 로봇 제어와 근본적으로 충돌한다는 문제를 timestep-invariance 특성(perception encoder ∂Enc/∂tₖ=0)으로 해결. Partitioned Attention(Static Prefix + Sliding History + Dynamic Suffix)으로 KV-cache를 오차 없이 재사용하고, AdaRMSNorm으로 BFloat16 스트리밍 안정성 확보, 비동기 파이프라인 + Future-State Predictor로 vision·policy를 병렬화. π0.5 기준 LIBERO에서 2.58× inference 가속, 반응 지연 −47~−54%, stall rate 100%→0%, 성공률 오히려 소폭 상승(long-horizon +3.6~+4.0pp). 실기 성공률 +11~+17pp.

1배경 · 문제의식

Flow-matching VLA(π0, π0.5 등)는 연속 action 궤적을 정밀하게 예측하지만, 매 step마다 timestep t를 글로벌 주입하는 구조라 KV 캐시가 무효화된다. 결과적으로 매 denoising step에서 O(N²) 전체 재계산을 하거나, 캐시 재사용을 강행해 수학적으로 잘못된 출력을 얻는 두 갈래 중 하나를 골라야 한다. 여기에 (a) 동기 실행이 필연적 100% stall을 유발하고 로봇이 chunk 사이마다 멈춤, (b) 초당 50회 이상 반복 추론에서 BFloat16 activation spike로 NaN/Inf가 자주 발생하는 문제가 겹친다.

Reflex의 관점: perception encoder는 denoising loop와 기능적으로 독립(∂Enc/∂tₖ=0)이라는 점을 활용하면, 관찰 인코딩과 timestep-의존 흐름 상태를 구조적으로 분리해 캐시를 안전하게 재사용할 수 있다.

2핵심 Contribution

  • Timestep-Invariance + Partitioned Attention — context를 Static Prefix(시스템 지시) / Sliding History(관찰 FIFO, N=10 프레임 ≈300ms) / Dynamic Suffix(흐름 상태·timestep 임베딩) 3구역으로 분할. Suffix만 O(1)로 재계산해 full-batch attention과 MSE=0.00 동일한 출력.
  • AdaRMSNorm — RMS는 FP32로, gating MLP는 BF16으로 유지. 조건 c=[tₖ, sₜ]에서 γ(c)=1+MLP(c)를 예측. BF16 baseline은 120~220 step에서 NaN 발생, AdaRMSNorm은 2,000+ step 안정(+0.4ms 오버헤드).
  • Async 파이프라인 + Future-State Predictor — Vision 스레드(관찰 인코딩)와 Policy 스레드(action 생성)를 분리. Predictor가 오래된 sensor 상태 sₜ 대신 마지막 명령 aₜᶜᵐᵈ ≈ ŝₜ₊Δ로 단기 dynamics 선형화. Adaptive Overlap Scheduling이 lookahead K=⌈τ̄ᵢₙf/τₛₜₑₚ⌉를 실시간 조정.

3Method

Partitioned Attention (핵심 수식)

Attn(x⁽ᵏ⁾) = Softmax(Q⁽ᵏ⁾[Kpin; Kslide(t); Kdyn(k)]ᵀ/√d) × [Vpin; Vslide(t); Vdyn(k)]
Pin(정적 지시)는 초기 1회 계산 후 영구 캐시. Slide(관찰 FIFO)는 매 프레임 push/pop. Dyn(흐름 상태·timestep)은 denoising cycle마다 리셋. 고정 관찰창 하에서 원본 full attention과 정확히 동일한 출력이 보장된다.

AdaRMSNorm — BF16 스트리밍 안정성

AdaRMSNorm(x, c) = x/RMS(x) ⊙ γ(c) where γ(c)=1+MLP(c), RMS는 FP32. "Robust Dtype Inference"가 초기화 시 output projection 계층을 probing해 하드웨어 precision을 판별한다. 이는 지연이 아닌 안정성을 겨냥한 컴포넌트.

Asynchronous Pipeline

Thread A(Vision)는 카메라 프레임을 계속 인코딩해 KV pair를 push, Thread B(Policy)는 최신 캐시를 조회해 action chunk를 denoising. Predictor: ŝₜ₊Δ ≈ aₜᶜᵐᵈ (Eq. 3). Scheduler(Alg. 1)는 현재 chunk가 (C−K) iteration 남았을 때 다음 inference를 launch해 계산·실행을 겹치게 한다.

5Ablation (주요 발견)

  • Partitioned Attention 정확성 (Table 4): Naive Caching MSE 1.42 / 성공률 12.5%, Full Re-comp 85.2%, Partitioned MSE 0.00 / 85.4% — 정확도 손실 없이 캐싱.
  • BF16 안정성 (Table 5): BF16 baseline 120~220 step에서 NaN, AdaRMSNorm은 2,000+ step 무 NaN(+0.4ms). LIBERO 300~500 step, Kinetix 200~400 step 커버.
  • 컨텍스트 K (Fig. 10): K=1은 최고 speedup이나 성공률 급락, K=10이 스위트스팟(2.58×, ~79%). K=50에서는 speedup<1×.
  • Predictor 강건성 (Fig. 9, Kinetix): 4 delay step에서 Reflex +11pp(predictor 무 대비), Async-Naive 대비 +22pp.
  • Chunk 크기 (Table 9): 기본 50이 최적 균형(2.58×, 19.1Hz, 79.5%). 100은 speedup 소폭↑나 응답성↓.
  • Operator Fusion (Fig. 11): FlashNorm(-2.7ms) + FusedAdaLN(-1.5ms) → action expert 지연 18% 감소. 20ms/step 예산 달성의 필수 요소.
  • Hardware (Table 8): RTX 3090에서도 2.47× — 아키텍처 세대 무관하게 가속 유지.
