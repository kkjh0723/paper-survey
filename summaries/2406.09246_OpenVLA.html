OpenVLA: An Open-Source Vision-Language-Action Model

Moo Jin Kim, Karl Pertsch, Siddharth Karamcheti 외 17인 · Stanford · UC Berkeley · Toyota Research Institute · Google DeepMind · MIT · Physical Intelligence · arXiv:2406.09246

arXiv abstract PDF 프로젝트 페이지 GitHub arXiv:2406.09246
VLA Open Source Open X-Embodiment LoRA Fine-tuning Robot Manipulation

제출: 2024년 6월 13일 (v1) · 최종 개정 2024년 9월 5일 (v3) · 7B 파라미터 · 백본 Llama 2 7B

한 줄 요약 (TL;DR)

인터넷 규모 비전-언어 데이터로 사전학습된 VLM 위에, Open X-Embodiment의 97만 개 실로봇 데모를 파인튜닝해 만든 7B 오픈소스 VLA. Llama 2 백본에 DINOv2 + SigLIP 융합 비전 인코더를 얹었다. 파라미터가 7배 적음에도 29개 태스크에서 폐쇄형 RT-2-X(55B) 대비 절대 성공률 +16.5%p(BridgeData V2 WidowX 74.4% vs 57.9%) 우위. 소비자용 GPU에서 LoRA 파인튜닝·양자화 배포가 가능하며, 코드·체크포인트를 전부 공개해 VLA 연구의 진입장벽을 크게 낮췄다.

1배경 · 문제의식

비전-언어 데이터와 다양한 로봇 데모를 결합해 사전학습한 대형 정책(policy)은 로봇에게 새 스킬을 가르치는 방식을 바꿀 잠재력을 지닌다. 그러나 기존 VLA(예: RT-2-X)에는 두 가지 근본적 문제가 있었다: (1) 폐쇄성 — 모델이 비공개이고 아키텍처·학습 절차·데이터 혼합이 알려지지 않아 커뮤니티가 확장·연구하기 어렵다; (2) 파인튜닝 비법 부재 — 새 로봇/환경/태스크에 효율적으로 적응시키는 방법이 정립돼 있지 않아 실사용 장벽이 높다. 저자들은 완전 공개된 범용 VLA와 효율적 적응 레시피를 제시하는 것을 목표로 한다.

2핵심 Contribution

  • OpenVLA: 완전 오픈소스 7B VLA — Open X-Embodiment의 97만 실로봇 데모로 학습. 모델 가중치·PyTorch 학습 코드·노트북을 모두 공개해 재현·확장이 가능.
  • SOTA 범용 조작 성능 — 7배 작은 파라미터로 폐쇄형 RT-2-X(55B) 대비 29개 태스크 평균 절대 성공률 +16.5%p 우위. 다중 태스크·다중 물체 일반화에서 강함.
  • 효율적 파인튜닝·배포 레시피 — LoRA로 신규 태스크에 파라미터의 1.4%(97.6M)만 학습, 양자화(4-bit)로 성능 저하 없이 소비자 GPU 배포. 풀 파인튜닝 대비 GPU 8배 절감.

3Method

백본 & 아키텍처

기반 VLM은 Prismatic-7B 계열 구조로, 언어 모델은 Llama 2 7B. 비전 인코더는 DINOv2 + SigLIP 두 인코더의 특징을 채널 방향으로 융합(fused)해 사용 — DINOv2의 공간/기하 정보와 SigLIP의 의미 정보를 결합한다. 입력 이미지는 224×224 해상도.

행동을 토큰으로

연속 로봇 행동(7-DoF end-effector delta)을 차원별로 256개 bin으로 이산화해 토큰화하고, Llama 토크나이저의 사용 빈도가 낮은 256개 토큰을 action token으로 재할당(overwrite)한다. VLA는 언어 지시 + 현재 관찰을 받아 다음 행동 토큰들을 자기회귀적으로 예측하는 표준 언어모델 학습(next-token prediction)으로 훈련된다.

5Ablation (주요 발견)

  • 비전 인코더 융합: DINOv2+SigLIP 융합이 단일 SigLIP 대비 다중 물체 언어 grounding 태스크에서 약 +10% 향상. 공간 정보(DINOv2)와 의미 정보(SigLIP)의 상보성.
  • 비전 인코더 파인튜닝: VLM 관행처럼 동결하지 않고 함께 학습하는 것이 VLA 성능에 결정적.
  • 이미지 해상도: 224×224 vs 384×384 성능 차이 없음. 고해상도는 학습 시간만 3배 증가 → 224로 충분.
  • 학습 epoch: 데이터셋을 27 epoch 반복해야 하며, action token 정확도가 95%를 넘을 때까지 성능이 계속 개선됨 — 통상 VLM 학습(1~2 epoch)보다 훨씬 많은 반복 필요.
  • 양자화: 4-bit 양자화로 메모리를 줄여도 성능 저하가 사실상 없음 → 실배포 친화적.
