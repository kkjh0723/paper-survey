Scaling Behavior Foundation Model for Humanoid Robots

Weishuai Zeng, Kangning Yin, Xiaojie Niu (공동 1저자) 외 15명 · CUHK · SJTU · ZJU · PKU · Tsinghua · Galbot · Shanghai AI Laboratory · 교신저자 Jiangmiao Pang, Jingbo Wang

arXiv abstract PDF 프로젝트 페이지 arXiv:2607.15163
Humanoid Behavior Foundation Model Motion Tracking Scaling Unitree G1

제출: 2026년 7월 16일 (v1) · 로봇 Unitree G1 (29 DoF) · IsaacLab 학습 · MuJoCo 평가

한 줄 요약 (TL;DR)

휴머노이드용 행동 기반 모델(Behavior Foundation Model, BFM)을 효과적으로 확장(scaling)하는 세 가지 핵심 요소 — (1) 전역 좌표계(global frame)에서 전신 궤적을 재현하는 motion-tracking 학습 패러다임, (2) on-policy 롤아웃 양 × 참조 동작 다양성의 시너지, (3) 확장성 높은 Humanoid Transformer 아키텍처 — 를 체계적으로 규명한 논문. 102M 프레임 규모 인간 동작 데이터로 PPO 학습, 기존 컨트롤러 대비 테스트셋 MPKPE(평균 키포인트 위치 오차)를 로컬 모드 10% 이상, 글로벌 모드 82% 감소. 단 3M 파라미터 모델이 GMT·TWIST·SONIC 등 기존 전신 컨트롤러를 성공률·추적정확도 모두에서 앞섬.

1배경 · 문제의식

휴머노이드 전신 제어(WBC)는 자연스러운 전신 협응, 실시간 정밀 반응, 다양한 환경에서의 강건한 일반화를 동시에 요구한다. 기존 방식은 태스크별 보상 설계에 의존해 범용성·일반화가 제한됐다. 이를 극복하려는 BFM(대규모 행동 데이터로 사전학습된 범용 컨트롤러)이 부상했으나, "BFM을 어떻게 확장하면 성능이 오르는가"에 대한 원리는 불분명했다. 선행 연구(SONIC 등)는 참조 동작 수·모델 크기·GPU 시간을 늘리는 예비적 증거만 제시했을 뿐, 학습 패러다임·데이터·아키텍처가 어떻게 맞물려야 확장이 효과적인지는 규명하지 못했다. 이 논문은 그 확장 레시피(scaling recipe)를 체계적으로 재검토한다.

2핵심 Contribution

  • 글로벌 프레임 통합 전신 추적 패러다임 — 다양한 제어 문제를 "전역 좌표계에서 통합 전신 궤적을 재현하는 문제"로 재정식화. 루트 위치를 무시하거나(예: 전진 vs 제자리걸음이 구분 불가) 루트–자세 추적을 분리하는 선행 설계의 행동 모호성·협응 붕괴 문제를 해소하고, 하나의 BFM으로 글로벌 제어와 로컬 제어를 모두 지원.
  • 데이터 확장의 재정의: 양 × 다양성 시너지 — PPO에서 실제 학습 데이터는 참조 동작 수가 아니라 환경 상호작용으로 얻는 on-policy 롤아웃임을 명확화. 데이터 양은 병렬 환경 수·롤아웃 horizon(width·depth) 확장으로, 다양성은 참조 동작 코퍼스 확장으로 얻으며, 둘의 동종/이종 확장(homogeneous / heterogeneous) 구분이 성능을 좌우함을 규명.
  • Humanoid Transformer 아키텍처 — MLP 백본을 대체하는 확장성 높은 트랜스포머. 별도 보조 손실 없이 구조화된 행동 latent 표현이 자연 발생하도록 설계(RMSNorm 기반 구면 latent 공간). 중간 크기 트랜스포머가 훨씬 큰 MLP에 필적/상회.

3Method

문제 정식화 (GCRL + Motion Tracking)

BFM 사전학습을 목표조건 강화학습(GCRL)으로 구성하고, 대리 태스크(proxy task)로 전신 motion tracking을 채택 — 보상은 참조 동작을 얼마나 충실히 따르는가로 정의된다. 행동 B는 태스크별 목표(goal state)를 배제한 고유수용성 상태·행동의 궤적으로 정의되어, 행동 학습을 제어 모드와 분리한다. 최적화는 PPO로 수행하나 정식화 자체는 알고리즘 불가지론적.

학습 패러다임 · 제어 인터페이스

비대칭 actor-critic: actor는 배포 가능한 고유수용성(각속도·투영중력·관절위치/속도)만, critic은 시뮬레이터 특권정보(링크 위치/자세/속도 등)를 사용. 제어 인터페이스는 루트 상대 직교공간의 마스킹된 전신 목표 자세로, 사전 정의된 마스크 집합에서 링크별 마스크를 샘플링해 8가지 제어 모드(Root / Bimanual / Root-and-Hand / End-Effector / Root-and-End-Effector / Upper-Body / Root-and-Upper-Body / Whole-Body)를 구성. critic에는 마스킹 없는 완전한 목표를 제공.

보상 핵심: 루트 위치 추적을 제거하거나(BeyondMimic 등) 루트–자세 추적을 분리하는 대신, 로봇이 참조 동작을 전역 좌표계의 통합 전신 궤적으로 재현하도록 요구. 루트 이동의 편차가 전신으로 전파되어 더 일관된 행동 지도(coherent guidance)를 제공한다.

Humanoid Transformer 아키텍처

고유수용성·목표·행동을 유한 시간 윈도로 확장하고 모달리티별 토크나이저로 인코딩. 상태·행동 토큰을 인터리브한 컨텍스트 뒤에 학습형 query 토큰을 붙여 self-attention으로 이력을 집계하고(query는 컨텍스트로부터 attend 받지 않음), 목표 상태는 시간 오프셋과 함께 cross-attention으로 주입. RMSNorm으로 목표 임베딩을 구면(hypersphere)에 정규화해 별도 정규화 손실 없이 구조화된 latent 공간이 자연 발생. actor는 미래 5프레임 {0..4} + [5,32]에서 무작위 1프레임의 stochastic future offset(배포 지연 완화), critic은 지수 간격 {0,1,2,4,8,16,32}을 사용.

5Ablation (스케일링 발견)

  • On-policy 데이터 확장(양): 병렬 GPU 수(width)와 롤아웃 horizon(depth)를 함께 키우면 대부분 실험에서 성능 향상, 최대 구성 64 GPU × horizon 64가 거의 모든 실험에서 최고. 단, 한쪽만 키우면 일관된 개선이 없음 — width·depth의 적절한 균형이 중요.
  • 참조 동작 확장(다양성): 데이터를 XXS→XS→S→M→L 중첩 부분집합으로 나눠 occupancy rate(행동 군집 점유율)로 다양성 측정: XXS 0.9365 → S 0.9365(거의 불변) → L 0.9995(급증). 동종 확장(XXS→S, 양만↑)은 미미한 개선, 이종 확장(S→L, 행동 커버리지↑)은 신규 동작(Ours)에서만 큰 이득. 즉 새 행동이 평가 대상과 관련될 때에만 효과적.
  • 아키텍처: Humanoid Transformer가 MLP보다 대체로 높은 성공률·낮은 오차. 중간 크기 트랜스포머가 훨씬 큰 MLP에 필적/상회하며, 더 키우면 수익 체감. 일부 제어 모드는 조기 포화(난이도 불균형·모드 간 결합).
  • Latent 공간: 보조 손실 없이도 지역성·전역 조직성·강건성을 갖춘 구조화 latent 자연 발생(표 2: latent에 노이즈 레벨 20까지 회전 섭동을 줘도 성공률·오차 유지). 모델을 키우면 여러 제어 모드의 latent 표현이 수렴.
