Harness VLA: Steering Frozen VLAs into Reliable Manipulation Primitives via Memory-Guided Agents

Yixian Zhang, Huanming Zhang, Feng Gao, Xiao Li 외 · Yu Wang · Wenbo Ding · Chao Yu · arXiv:2607.08448

arXiv abstract PDF 프로젝트 페이지 arXiv:2607.08448
Frozen VLA Agentic Planner Primitive Library Memory Manipulation

제출: 2026년 7월 9일 · 백본 π0.5-SFT / RLDX-1 / LingBot-VLA (모두 동결) · 플래너: Codex, Claude Code

한 줄 요약 (TL;DR)

사전학습된 VLA를 파인튜닝하지 않고, LLM 기반 에이전틱 플래너 + 고정 primitive 라이브러리(analytic 6개 + vla_act 1개) + 성공/실패 메모리로 감싸 컨택트-리치 조작을 재시도 가능한 primitive로 노출한다. 플래너가 언어 지시·씬 그라운딩을 담당하고 VLA는 국소 접촉 구간만 맡음. LIBERO-Pro +38.6pt, RoboCasa365 composite-seen +25.4pt, RoboTwin C2R 58.4% zero-shot — 모두 VLA 가중치 수정 없이 달성.

1배경 · 문제의식

범용 VLA는 언어 이해와 컨택트-리치 조작을 하나의 신경망 안에 얽어 넣는다. 결과적으로 (i) 지시가 조금만 바뀌면 학습된 궤적을 재생하는 경향(instruction rigidity), (ii) 물체 배치·시야 변화에 취약, (iii) 실패 시 재시도가 아니라 그대로 실패 모드로 수렴. 저자들은 VLA를 다시 학습하는 대신 그 위에 메모리와 primitive를 얹은 에이전트 레이어를 두어, 계획·재그라운딩은 심볼릭 수준에서, 접촉 실행은 프로즌 VLA에게 각각 맡기는 분리 구조를 제안한다.

2핵심 Contribution

  • Frozen VLA를 primitive로 노출. vla_act(target, staging)이라는 재시도 가능한 접촉 primitive를 move_to · rotate_wrist · set_gripper · release · navigate_to 같은 analytic primitive와 나란히 배치, 플래너가 순서·파라미터를 결정.
  • 메모리 이중화. Task Specific Memory는 성공한 명령 트레이스(JSONL), Global Memory는 재사용 가능한 성공 규칙과 실패 모델을 저장. 파라미터화된 공간 쿼리로 재사용성을 확보.
  • 퍼터베이션 강건성. LIBERO-Pro의 지시 재지정·위치 스왑에서 이전 SOTA(RATS 43.8%) 대비 +38.6pt, RoboCasa365 composite-seen에서 RLDX-1 대비 +25.4pt, RoboTwin C2R zero-shot에서 58.4% — 모두 VLA 가중치 수정 없이 달성.

3Method

Primitive Library

6개 analytic primitive(move_to, rotate_wrist, set_gripper, release, navigate_to + 모바일 베이스 확장 시 총 9개) + 1개 학습형 vla_act. Analytic primitive는 perception-conditioned staging·transport·release를 결정론적으로 처리하고, vla_act는 국소 컨택트-리치 구간(파지·삽입·꼬집기 등)만 담당.

Agentic Planner

Codex 또는 Claude Code(CC)를 LLM 플래너로 사용. REPL-style harness가 턴 기반으로 (관찰 → 플래너 결정 → primitive 실행 → 결과) 루프를 돌린다. 부트스트래핑 단계에서는 reset primitive와 넉넉한 예산이 허용되나, 평가 단계에서는 reset 없이 엄격한 스텝 예산으로만 평가.

Memory Modules

Task Specific Memory
성공 실행 트레이스(JSONL) — 동일 태스크 내 few-shot 예시로 활용
Global Memory
재사용 가능한 성공 규칙 + 실패 모델(교차 태스크)
쿼리 형식
parameterized spatial queries — 물체 위치·방향 조건으로 검색 가능

Adaptive VLA Invocation

플래너가 vla_act를 호출하고 실패 시 재그라운딩·재스테이징 후 재호출. 태스크당 최대 호출 수를 늘려도 초기 몇 번 후 성공률이 saturate하는 sparse-but-effective retry 패턴을 보인다(그림 4).

5Ablation / 핵심 발견

  • Semantic Re-grounding. 플래너가 언어 해석·씬 바인딩을 심볼 수준에서 담당하면, 프로즌 VLA는 접촉 실행에만 집중하고도 지시-조건 행동을 회복한다.
  • Adaptive VLA Invocation. 초기 몇 번의 vla_act 호출로 성공률이 급상승 후 saturate — 재시도가 희소하지만 효과적. 무한 호출이 필요하지 않다.
  • Analytic Decomposition. 태스크 완료 원인(그림 6): LIBERO 변형은 대부분 analytic primitive(파랑)로 마무리, RoboCasa365·양팔 태스크는 VLA primitive(주황)로 마무리. 벤치마크 특성에 따라 분업이 자동 최적화됨.
  • 플래너 인스턴스 비교. Codex와 Claude Code는 상보적 — LIBERO-Pro는 CC가 우세(82.4 vs 72.1), RoboCasa365 Composite-Seen은 Codex가 우세(56.3 vs 47.5).
