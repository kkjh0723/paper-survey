RoboHarness: Memory-Driven Orchestration of Heterogeneous Robot Policies for Long-Horizon Planning

Jinbang Huang, Yuanzhao Hu, Zhiyuan Li, Ran Qi, Yixin Xiao, Zhanguang Zhang, Mark Coates, Tongtong Cao, Yingxue Zhang (McGill/Huawei Noah's Ark 계열)

arXiv abstract PDF HTML arXiv:2607.18060
Long-Horizon Memory Bridge VLA · RL · TAMP Zero-shot Orchestration Robustness

제출: 2026년 7월 20일 · 3개 벤치마크 + 500 커스텀 태스크 + 135 실로봇 시행

한 줄 요약 (TL;DR)

단일 policy로 커버 불가능한 long-horizon 태스크를 위해, 독립 개발된 VLA·RL·TAMP·MPC·Nav 등 이종(heterogeneous) 정책들을 재사용 스킬로 묶고 다중모달 실행 메모리로 능력 경계를 특성화·라우팅하는 오케스트레이터. 핵심은 Memory Bridge — 과거 궤적으로 in-distribution 상태 영역을 추정해 정책 간 재학습 없이(handoff-time) 안정적으로 넘겨준다. LIBERO 평균 98.7, LIBERO-Plus(교란 평균) 93.2(π0.5 85.7, OpenVLA-OFT 67.9), 장기 태스크 LIBERO-LoHo 성공률 95.2(π0.5 단독 6.4), 실로봇 조립 총 86.7%.

1배경 · 문제의식

실세계 로봇 장기 태스크는 조작·이동·계획·인식 등 다양한 능력이 필요하고, 어떤 단일 정책도 이를 신뢰성 있게 커버하지 못한다. 실무에서는 이미 잘 학습된 VLA(예: π0), RL 정책, TAMP 시스템, MPC 컨트롤러 등이 독립적으로 존재하지만, 이들을 재훈련 없이 조합해 하나의 긴 시나리오를 풀려면 두 가지 난제가 있다: (1) 각 정책의 능력 경계가 명세되어 있지 않아 언제 어떤 정책을 호출해야 하는지 알 수 없고, (2) 정책을 갈아탈 때 후속 정책의 분포 밖(out-of-distribution) 상태로 handoff되어 실패한다. RoboHarness는 이 두 문제를 메모리 기반 오케스트레이션으로 정면 돌파한다.

2핵심 Contribution

  • Capability-aware Orchestration — 다중모달 실행 메모리 + 온라인 증거(uncertainty·visual-context·semantic-context·state-policy compatibility·input-quality)를 결합해 각 정책의 능력 경계를 자동 특성화하고 태스크 세그먼트를 라우팅한다.
  • Memory Bridge — 과거 궤적으로 anchor 세트를 구성 → 공간 상태 분포를 추정 → handoff 후보 상태 점수화 → bridge trajectory를 생성해 후속 정책의 in-distribution 영역으로 상태를 옮긴다. 재학습 불필요.
  • 이종 정책의 통합 프레임워크 — VLA, RL, TAMP, MPC, Nav 등 인터페이스가 서로 다른 컨트롤러를 스킬로 wrap해 하나의 파이프라인에서 협업. 실패 시에만 self-evolution(파라미터/메타데이터/하네스 refinement) 트리거.

Understanding Skills
uncertainty 평가, visual/semantic context 매칭, state-policy 호환도, 입력 품질 — 결정에 필요한 정보를 추출

Self-Evolution Skills
policy adaptation·harness refinement·parameter tuning·metadata update — 실행 실패 시에만 발동(매 rollout 이후가 아님)

Memory Bridge 파이프라인

정책 A → 정책 B로 넘어가는 순간, 다음 4단계로 handoff를 안정화한다:

  • (a) Anchor 검색 — 정책 B의 성공 궤적들 중 현재 컨텍스트에 부합하는 anchor trajectory를 검색.
  • (b) 분포 추정 — anchor로부터 정책 B가 잘 작동하는 공간 상태 분포를 구성.
  • (c) 후보 점수화 — 현재 상태 근방의 handoff 후보 상태들을 in-distribution 정도로 점수화.
  • (d) Bridge trajectory 생성 — 최고 점수 상태로 로봇을 옮기는 짧은 bridge 궤적을 생성해 실행 후 정책 B에 제어권 전달.
핵심 통찰: 정책 간 실패는 대개 "능력 부족"이 아니라 "handoff 시점의 상태가 후속 정책의 학습 분포에서 벗어난 것". Memory Bridge는 이를 재훈련 없이 상태 조작으로 해결한다.

온라인 능력 경계 학습

초기에는 각 정책의 metadata(문서상 스펙)로 라우팅을 시작하지만, 실행 성공/실패 시 다중모달 메모리에 (관찰, 명령, 결과)를 저장하고 실패 사례로부터 정책별 능력 경계를 refine한다. 오케스트레이터 자체는 파운데이션 모델을 그대로 활용(LLM/VLM), 학습되는 것은 메모리와 하네스 메타데이터뿐이다.

5Ablation (주요 발견)

  • Understanding Skills 제거 — 가장 큰 성능 저하. 라우팅 근거가 사라지면 오케스트레이션 자체가 무의미해짐.
  • Self-Evolution Skills 제거 — 능력 경계 refinement 불가. 시간이 지나도 오분류가 축적됨.
  • Memory Bridge 제거 — "정확한 상위 오케스트레이션"이 "신뢰 가능한 실행"으로 이어지지 않음. 정책 선택은 맞지만 handoff에서 실패.
  • 단일 정책 변형 — 각 태스크에서 자기 능력에 정합한 세그먼트만 해결. 전체 성공 불가.
