Retrieve, Don't Retrain: Extending Vision-Language-Action Models to New Tasks at Test Time

Jeongeun Park, Juhan Park, Taekyung Kim, Sungjoon Choi, Dongyoon Han, Sangdoo Yun · NAVER AI Lab · Korea University

arXiv abstract PDF 프로젝트 페이지 arXiv:2606.15631
VLA Retrieval Test-Time Adaptation World Model Cross-Embodiment

제출: 2026년 6월 14일 (v1) · 백본 Cosmos Predict2.5 (2B)

한 줄 요약 (TL;DR)

새 태스크를 배우려고 재학습(retrain)하지 말고 검색(retrieve)하라. 정책을 동결한 채, 값싼 pool-embodiment(예: 인간 손 영상) 데모를 검색 풀에 추가하는 것만으로 새 태스크를 흡수하는 테스트타임 태스크 적응 패러다임. 파인튜닝은 새 embodiment를 받을 때만 필요하고, 새 태스크마다는 불필요. 검색이 대략적 고수준 동작을, 정책은 embodiment 특화 잔차(residual) 보정만 학습. 인간 손 데모는 로봇 텔레옵 대비 ~18배 저렴, WAM 태스크별 파인튜닝 ~24 GPU-hours/task를 인덱싱으로 대체. PushT 미학습 각도 34.9%(vs 6.0%), RoboTwin 2.0 미학습 태스크 31.5%(vs 26.0%).

1배경 · 문제의식

교차 embodiment 모방학습에서 query(배포 대상, 예: 로봇 팔)와 pool(더 값싼 embodiment, 예: 인간 손, 배포되지 않음)의 두 축을 다룬다. 전이는 두 가정에 기댄다: (1) 공유 상태/행동 표현 — SE(3) end-effector pose + 그리퍼 신호; (2) 궤적 수준에서 의미적으로 유사한 동작. 기존엔 새 태스크마다 텔레옵 수집 + 정책 파인튜닝(Cosmos Policy 기준 ~24 GPU-hours/task)이 필요했다.

2핵심 Contribution

  • 재학습 없는 테스트타임 태스크 적응: 값싼 pool-embodiment 데모를 검색 풀에 추가하는 것만으로 새 태스크 흡수. 파인튜닝은 새 embodiment에만 필요.
  • RECAP (Retrieval-Conditioned Action Policy): world-action model(WAM, Cosmos Policy) 위에 세운 검색 조건부 잔차 정책. 검색이 대략적 동작/진행을, 정책은 embodiment 특화 보정(residual)만 학습. WAM의 미래-이미지 목적은 검색과 결합될 때만 유효.
  • 비용 논거: 인간 손 데모는 텔레옵 대비 ~18배 저렴, 태스크별 파인튜닝 ~24 GPU-hours를 인덱싱으로 대체.
  • 실증적 우위: PushT, RoboTwin 2.0, 실로봇에서 교차 embodiment 베이스라인 상회 + 파라미터 갱신 없이 풀 확장에 따른 단조 성능 향상.

3Method

검색 증강 WAM

Cosmos Policy(query 행동 + 미래 이미지 관찰을 하나의 denoised 비디오 시퀀스로 출력) 기반. RECAP은 검색된 pool 청크를 조건으로 추가: πθ(squery, (spool, apool)) ↦ âquery, ŝquery. 검색 청크+관측 query 프레임을 clean latent로 인코딩해 시간축 앞에 prepend, query 미래는 noise에서 denoise. 언어 지시는 cross-attention으로 진입 — 구조 변경 없이 표준 I2V 조건화를 clean 상태-행동 서브시퀀스로 확장.

잔차 행동 파라미터화

âquery = apool + Δa — action latent은 "query 행동이 pool과 어떻게 다른가"(embodiment gap 보정)만 인코딩. 이 차이는 행동 레이블엔 약하게, 픽셀엔 뚜렷하게 나타나므로 미래 상태 예측(state prediction)이 이를 학습할 dense한 시각 신호를 공급. 공동 목적: L = λ·Lact + Lstate (단일 flow-matching 손실).

2단계 검색

Stage 1(궤적 prefilter): 초기 프레임 기술자 ψ0 = 언어 목표 임베딩 + 초기 관련 물체 위치(SAM 3) + 초기 proprioception 하에 query에 가장 가까운 top-K 궤적. Stage 2(서브프레임 매칭): 후보 내에서 물체 pose·proprioception·(학습 시에만) 다가올 행동 청크의 가중 L2 + DINOv3 이미지 특징 코사인 거리로 매 스텝 최적 t' 선택. H 길이 청크를 가져와 잔차 행동 예측, 앞 K개 실행 후 K만큼 전진해 재검색. 추론 시 Dpooltest가 활성 풀을 대체·재인덱싱하며 매 스텝 재검색 → 세션 내 풀이 파라미터 변경 없이 성장 가능.

핵심 ablation

  • 백본 비교(그림 5a): π0.5(행동만) 6.6%→25.1%, Cosmos(WAM) 6.0%→34.9% — WAM이 검색으로 더 큰 이득.
  • 공동학습 역할(그림 5b): 미래-이미지 예측 제거 시 34.9%→27.4%(검색 π0.5 수준). 검색 없이는 미래 예측이 중립/유해(8.3%→6.0%).
  • 2×2(표 3): residual+next-state 34.9 vs residual만 27.4 vs absolute+next-state 27.4 — next-state의 이득은 WAM×검색 상호작용에 특화(+7.4 under residual, 비검색 regime에선 −2.3).

실로봇 (pool=인간 손, target=텔레옵 로봇, 그림 8b)

open-cabinet 25 페어 데모로 파인튜닝 후 동결. held-out 태스크당 인간 손 데모 10개만 풀에 추가. Open 90%/90%, Place 10%→80%, Close 0%→30% — 베이스라인은 held-out에서 학습된 open 동작으로 붕괴.

5메커니즘 분석 (해석성)

디코더의 2단계 라우팅 관찰: L10 "intake"(검색된 태스크 영역에 주의, ret-1 T-블록 5.1×) + L15 "commit"(정책 자신이 예측한 종료 pose에 주의, 3.9×). 두 축 모두 검색이 없으면 사라짐(ROI 비율 ≈ 1.0). L10 causal 마스킹 시 성공률 20~80pp 하락, 실패 사례에서 L10 마스킹이 0%→20~40% 회생시키기도 함.
