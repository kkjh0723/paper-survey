ASPIRE: Agentic Skill Programming through Iterative Robot Exploration

Runyu Lu, Yubo Wu, Ethan Kou (공동 1저자) 외 · NVIDIA · UMich · UIUC · UC Berkeley · CMU · 프로젝트 리드 Yuke Zhu, Linxi "Jim" Fan, Guanzhi Wang

발표: 2026년 6월 30일 · © 2026 NVIDIA · 자기개선형(self-improving) 로봇 코딩 에이전트

한 줄 요약 (TL;DR)

로봇 프로그램을 스스로 작성·디버깅·개선하는 자기개선형 에이전트 시스템. 기존 로봇 코딩 에이전트는 (1) "성공/실패"만 알려주는 거친 피드백과 (2) 태스크마다 경험이 휘발되는 문제를 가진다. Aspire는 ① 프리미티브 단위의 멀티모달 실행 추적 엔진 → ② 검증된 수정을 재사용 가능한 스킬로 축적하는 스킬 라이브러리 → ③ 진화적 탐색(evolutionary search)의 세 축을 하나의 열린 학습 루프로 결합한다. 결과: LIBERO-Pro 최대 +77점, Robosuite 양팔 핸드오버 +72점, BEHAVIOR-1K 최대 +32점, 그리고 실제 로봇으로의 스킬 전이(토큰 사용량 최대 ~10× 절감)까지 초기 증거 제시.

1배경 · 문제의식

로봇 프로그래밍은 멀티모달 지각, 복잡한 접촉 동역학, 다양한 환경·실패 구성을 동시에 다뤄야 한다. 최근 LLM 기반 로봇 코딩 에이전트가 등장했지만 두 가지 근본적 한계가 있다.

① 거친 피드백(coarse feedback): 일반적인 실행 환경은 태스크 수준의 성공/실패 신호만 준다. 롤아웃이 실패해도 원인이 지각 오류인지, 불안정한 그리핑인지, 계획 오류인지, 복구 실패인지 알 수 없다. 세밀한 진단 추적이 없으면 에이전트는 실패 지점을 특정하거나 알맞은 수정 전략을 고를 수 없다.

② 경험 미축적: 각 태스크에서 찾아낸 수정·복구 전략이 태스크가 끝나면 버려진다. 그래서 "100번째 태스크를 푸는 에이전트가 첫 태스크를 푸는 에이전트보다 조금도 더 노련하지 않다." 반면 인간 로봇 엔지니어는 디버깅 경험을 전이 가능한 지식으로 복리처럼 축적한다.

2핵심 Contribution

  • 닫힌 루프 로봇 실행 엔진 — 롤아웃 단위가 아닌 프리미티브(primitive) 단위의 멀티모달 추적(호출된 API·입출력·반환 상태·RGB 키프레임·오버레이·그립 후보·물체 자세·모션플래닝 결과)을 기록해 실패 원인을 정확히 귀속(attribution)하고, 수정안을 닫힌 루프로 검증.
  • 지속적으로 확장되는 스킬 라이브러리 — 검증된 수정을 재사용·전이 가능한 in-context 스킬로 증류(distill)하여 후속 에이전트가 검색해 활용. 교차 태스크 및 교차 임바디먼트(sim→real) 전이 지원.
  • 진화적 탐색(Evolutionary Search) — 다양한 후보 프로그램을 생성해 단일 궤적 미세수정의 국소 함정을 벗어나 넓게 탐색.
  • 결과 — 세 축이 하나의 개방형(open-ended) 학습 루프를 이뤄 태스크를 거듭할수록 능력이 복리로 성장. 다수 벤치마크에서 큰 폭 개선 + 실제 로봇 전이 증거.

3Method

전체 구조 — Coordinator–Actor

중앙 coordinator가 공유 스킬 라이브러리를 관리하고, 태스크마다 actor 코딩 에이전트를 병렬로 배치한다. actor들은 전체 대화 이력이나 원본 궤적을 공유하지 않고, 오직 증류된 스킬만 주고받는다. 덕분에 각 actor의 컨텍스트는 태스크 명세·현재 프로그램·구조화된 실패 추적에 집중된 채 유지된다.

① 로봇 실행 엔진 (§2.1)

각 프리미티브 호출 직전/직후 프레임과 오버레이·반환값을 저장한다(전체 영상이 아니라 키프레임만). 예시(BEHAVIOR-1K 라디오 집기): navigate_to_pose가 반복적으로 PLANNING_ERROR를 반환 → 목표가 테이블 모서리에서 약 20cm 떨어져 cuRobo 충돌 회피가 발동한 것이 원인. 에이전트가 다각도 접근(Multi-Angle Approach) 루틴을 작성 → 스킬로 승격.

② 스킬 라이브러리 (§2.2)

미리 규정되지 않은 이질적 분류체계(위치추정 휴리스틱·지각 프롬프트·그리핑 제약·내비게이션 복구·모션 프리미티브·장면 이해 루틴·디버깅 워크플로 등). 각 스킬 = 실패 시그니처 + 적용 조건 + 수정 전략 + (선택)코드 스케치. coordinator가 actor의 발견을 감사하고 API 정책 준수를 확인해 디버그 검증을 통과한 재사용 가능한 수정만 등재.

③ 진화적 탐색 (§2.3, Algorithm 1)

매 라운드마다 상위 성능 프로그램(Top3)과 실패 추적을 조건으로 K개의 후보를 제안·실행하고, 다음 라운드는 최고 후보 + 잔여 실패를 조건으로 삼는다. 후보가 디버그 구성을 풀거나 예산이 소진되면 종료. 태스크별 지속 분석 문서(장면 설명·가설 목록·배제/차단된 방향 원장)를 유지.

중요한 제약(sim→real 전이를 위한 설계): 에이전트는 시뮬레이터의 ground truth 접근이 엄격히 금지된다(env.handle.env.sim, sim.data.body_xpos 등 금지, .bddl/.xml/.urdf 에셋 파일 읽기 금지). 오직 관찰과 추적만으로 진단해야 하므로 획득한 스킬이 실제 로봇으로 전이된다.

4실험 결과

헤드라인: LIBERO-Pro 최대 +77점 · Robosuite 양팔 핸드오버 +72점(0.20→0.92) · BEHAVIOR-1K 최대 +32점 · LIBERO-Pro Long 제로샷 31%(기존 4% 대비) · 실제 로봇 전이 시 토큰 최대 ~10× 절감.

세팅: 시뮬레이션 = Claude Opus 4.6 / CaP-X / MuJoCo, Franka 팔. 실제 로봇 = Codex GPT-5.5, 양팔 YAM. 디버그/평가 시드는 분리(disjoint). 주 베이스라인 CaP-Agent0, 이외 OpenVLA·π0·π0.5·인간 전문가.

Ablation (구성요소 기여)

로봇 실행 엔진만으로 macro 평균 성공률 14% → 62%, 여기에 진화적 탐색을 더하면 72%. 탐색은 초반에 꾸준히 이득을 주고 이후 수확 체감.

5한계 · Future Work

  • 아직 완전 자율 실세계 평생학습기는 아님 — 견고한 성공 판정, 안전한 리셋, 안전 모니터링, 캘리브레이션 유지가 필요.
  • 동결된 프런티어 LLM(Opus 4.6, 1M 컨텍스트)에 의존 — 더 작고 약한 LLM은 미검증.
  • 사전 정의된 지각·계획·제어 API에 묶여 있음 — 향후 에이전트가 새 프리미티브를 안전하게 제안·검증하는 방향.
  • 스킬 라이브러리의 장기 메모리 관리가 미흡 — 낡거나 중복·오도하는 항목이 비단조 전이를 유발할 수 있어 검색·프루닝·랭킹·재검증 개선 필요.
  • 디버그+진화탐색 루프가 계산 집약적(태스크당 다수 LLM 호출/롤아웃) — 저렴한 추론, 표본 효율적 탐색, 수정 재사용 강화가 과제.
