MimicGen: A Data Generation System for Scalable Robot Learning using Human Demonstrations

Ajay Mandlekar, Soroush Nasiriany, Bowen Wen, Iretiayo Akinola, Yashraj Narang, Linxi Fan, Yuke Zhu, Dieter Fox · NVIDIA · UT Austin

arXiv abstract PDF 프로젝트 페이지 GitHub arXiv:2310.17596 · CoRL 2023
Data Generation Imitation Learning Behavioral Cloning Manipulation Sim-to-Sim

제출: 2023년 10월 26일 · CoRL 2023 · 18개 태스크 · 200개 인간 데모 → 50K+ 합성 데모

한 줄 요약 (TL;DR)

단 ~200개 인간 데모를 객체 중심(object-centric) subtask 단위로 분해·변환·재실행해 50,000개+ 합성 데모를 자동 생성하는 시스템. 각 subtask에서 현재 물체 pose에 맞게 원본 end-effector 궤적을 SE(3) 변환하고 선형 보간으로 연결·실행해 데이터를 확장. 다양한 초기 상태·물체 인스턴스·로봇 팔(Panda/Sawyer/IIWA/UR5e)에서 BC 정책이 강한 성능(Nut-and-Bolt Assembly 96%, Coffee 93%, Stack Three 91%). MimicGen 200개 ≈ 인간 200개(79% vs 84% on Square D0). 이후 이 데이터·프레임워크는 π0·RoboCasa 등 후속 대규모 파운데이션 학습의 데이터 파이프라인 표준이 됨.

1배경 · 문제의식

Behavioral cloning으로 실용 조작 정책을 얻으려면 수천~수만 개의 인간 데모가 필요하지만 수집 비용이 극도로 비싸다. 시뮬레이션에선 초기 상태 분포가 넓어질수록 데모가 더 필요해 확장이 힘들다. MimicGen은 이 병목을 우회하는 자동 데이터 생성 시스템으로, 소수의 인간 데모를 다양한 새 컨텍스트(scene, object 위치, 로봇 팔)에 adapt해 대규모 데이터셋을 만든다.

2핵심 Contribution

  • MimicGen 시스템 — 인간 데모를 object-centric subtask로 분해하고, 각 subtask에서 target object pose에 맞춰 원본 궤적을 SE(3) 변환한 뒤 실행. 실패한 궤적은 폐기하고 성공만 데이터에 포함.
  • 18개 태스크 · 50K+ 데모 생성 — Stack/Square/Coffee/Threading/Assembly류·Kitchen·Mobile Kitchen·Nut-and-Bolt/Gear/Frame Assembly 등 long-horizon·high-precision 태스크 커버.
  • Cross-arm / cross-scene 일반화 — 동일 인간 데모(대개 Panda로 수집)에서 Sawyer, IIWA, UR5e용 데이터 자동 생성. 다양한 reset 분포·물체 인스턴스에서도 데이터 확장.
  • MimicGen ≈ 인간 데모 — 200 MimicGen 데모(합성)로 학습한 BC 정책이 200 인간 데모 학습 정책과 유사(예: Square D0 79% vs 84%). 자동 생성이 인간 수집 대안으로 경쟁력.
  • 오픈소스 · 후속 영향 — robomimic·robosuite 스택 위 코드/데이터셋 공개. 이후 RoboCasa365, π0/π0.5 등 대규모 학습의 표준 데이터 파이프라인으로 채택.

3Method

3.1 Subtask 분해 (object-centric)

인간 데모를 subtask termination signal(예: "gripper closed on object", "object placed")로 분해해 segments로 만든다. 각 segment는 시점별 end-effector pose · object pose · gripper action · termination flag를 담은 DatagenInfo 시퀀스. 각 subtask는 참조 물체 하나에 결부되어(object-centric) 그 물체의 pose 프레임에서 궤적을 표현할 수 있게 된다.

3.2 Source segment 선택

SelectionStrategy가 새 subtask마다 참고할 원본 segment를 고른다. 대표 전략인 NearestNeighborObjectStrategy는 "현재 scene의 물체 pose와 가장 가까운 pose를 가진 원본 데모"를 선택 (position·rotation weighting + k-NN). 매 subtask마다 다시 고르거나(하이브리드), 한 생성 시도 내내 유지할 수도 있다.

3.3 Pose 변환 (transform)

선택된 source segment를 현재 물체 pose로 재정렬. transform_source_data_segment_using_object_pose가 원본 물체 pose에서 목표 물체 pose로의 SE(3) 변환을 원본 EE 궤적에 적용해, 물체 상대적으로는 "같은" 조작이지만 새 위치·자세에 맞는 새 EE 궤적을 얻는다.

3.4 실행 (transform-then-linear-interpolate)

WaypointTrajectory가 (1) 현재 EE pose에서 (2) 변환된 첫 waypoint까지 선형 보간(연결 구간)한 뒤, (3) 변환된 subtask 궤적을 그대로 재생. execute가 각 waypoint를 action으로 변환해 시뮬 환경에 인가하고, 관찰·grip 성공 여부를 저장. 매 subtask 성공을 검증해 실패 시 폐기 → 성공 궤적만 데이터셋에 포함.

핵심 아이디어: "물체 프레임에서의 조작 궤적은 물체 위치에 (거의) 불변"이라는 가정 하에, 원본 인간 데모를 충분히 유사한 물체 pose로 옮겨 재사용. 이 가정이 깨질 때(장애물·복잡 접근 경로)엔 실패율이 높아지지만, 대량 시도 후 성공만 채택하므로 데이터 품질은 유지된다.

Cross-arm / Cross-scene 확장

  • 같은 소스 인간 데모(Panda) → Sawyer, IIWA, UR5e에서 새로 데모 수집 없이 훈련 데이터 생성.
  • 여러 reset 분포(D0/D1/D2, 초기 상태 범위 확장)에서 태스크당 1,000 데모 자동 생성.
  • Stack, Coffee 등 실 로봇에서도 sim→real 후 정책 실행 성공을 시연.

5Ablation (주요 발견)

  • 데이터 품질 vs 양: 소수 데이터 regime에서는 인간 데이터가 우세하지만, 대규모 regime에서는 MimicGen 데이터 품질 저하가 성능에 큰 영향을 주지 않음. → "많이 뽑아 성공만 남기는" 전략이 통함.
  • Source 데모 개수: 태스크당 ~10개 인간 데모만 있어도 대부분 태스크에서 1,000개 이상 성공 데이터 생성 가능.
  • Cross-embodiment: Panda 데모로 만든 궤적이 Sawyer/IIWA/UR5e에서도 잘 재생됨(EE-space 궤적이므로 로봇 kinematic 차이에 상대적 강건).
  • 실패 요인: subtask segmentation이 잘못되거나 물체 접근 경로에 장애물이 있으면 성공률 급락 — 이때 대량 시도 후 성공만 채택하는 방식으로 우회.
