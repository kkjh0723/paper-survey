한 줄 요약 (TL;DR)

사전학습된 플로우 매칭 기반 생성형 로봇 정책(VLA·WAM·diffusion policy)을 가중치 수정 없이 소수의 인간 개입만으로 배포 현장 적응시키는 DAgger 계열 기법. 핵심은 action inversion — 사람이 시연한 보정 액션 a*를 프로즌 정책이 그것을 생성하도록 하는 latent noise w*로 per-step 고정점 반복(M=5)로 뒤집어내고, 작은 MLP πw(s)가 표준 노이즈 대신 이 w*를 예측하도록 학습. MetaWorld 평균 SR +0.25, 실제 하드웨어 Toolbox Packing +0.67, 그리고 미개입 held-out 태스크 성능은 SFT 대비 −0.94 → FlowDAgger −0.08로 사전지식 파괴를 사실상 없앰.

1배경 · 문제의식

π0·π0.5 같은 대형 VLA는 배포 시 예상 못 한 실패 모드를 만난다. 대규모 재수집 파인튜닝(SFT)은 held-out 태스크를 망가뜨리고(0.96 → 0.02), LoRA·잔차(residual) 정책도 사전지식 유지에 실패한다. 온폴리시 RL은 물리 안전·샘플 효율 문제로 현장에서 비현실적이다. FlowDAgger는 이 갭을 인간 개입 수십 회 규모의 데이터로, 프로즌 정책을 훼손하지 않으면서 메우려 한다.

2핵심 Contribution

  • Action Inversion — per-step 고정점 반복. 플로우 매칭 정책의 K≈10 Euler 스텝을 각 스텝에서 M=5 반복으로 역산해, 사람이 준 보정 액션 a*와 정확히 대응되는 latent noise w*를 ~0.00168 MSE로 복원(Euler-reverse 0.0329 대비 20×↓, trajectory-level 0.0228 대비 14×↓).
  • Latent Noise Policy로 프로즌 정책 조종. 표준 가우시안 노이즈 대신 상태 조건부 πw(s)가 만든 노이즈를 초기값으로 사용 → base 정책의 가중치는 불변. 자율 성공 rollout과 반전된 개입을 배치당 1:1로 섞어 정규화.
  • 아키텍처 무관 확장. flow-matching action head(π0.5), 비디오 diffusion world-action model(Cosmos-Policy), DiT+DCT(Gr00t N1.7), vanilla DDPM(robomimic)까지 동일 원리로 적용. WAM에서는 공동 latent 프로세스를 반전하되 action 프레임만 섭동.

3Method

Per-step 고정점 반전

플로우 매칭 정책은 노이즈 xK~N(0,I)에서 시작해 xk = xk+1 − Δt·vθ(xk, tk, s)로 K≈10 스텝 Euler 적분해 액션 x0를 만든다. 인간 보정 a*가 주어지면 x0=a*로 고정하고 위 재귀식을 뒤에서 앞으로 M=5번 fixed-point iteration으로 풀어 각 스텝의 xk를 얻는다. 이는 trajectory 전체를 한꺼번에 뒤집는 방식보다 정확·경제적(0.00168 vs 0.0228 MSE, 456 ms vs 1662 ms).

Noise Policy 학습

L(φ) = E(s,w*)~D ‖πwφ(s) − w*‖²로 소형 MLP만 지도학습. 데이터셋 D는 반전된 개입 + 자율 성공 rollout을 배치당 동일 비율로 유지해, 개입이 없는 상태에서 base 성능이 붕괴하지 않도록 한다.

5Ablation

  • 반전 절차 비교(Assembly): Per-step FP(M=5) 0.00168 MSE / 456ms / SR 0.87 · Trajectory FP 0.0228 / 1662ms / 0.77 · Euler reverse 0.0329 / 315ms / 0.59. Per-step FP가 정확도·속도·SR 모두 우세.
  • 반복 횟수 M: Action MSE 0.00397(M=3) → 0.00168(M=5) → 0.00147(M=10). M=5에서 saturate.
  • 자율 성공 rollout 비율: 개입만 학습하면 sparse-correction 상태에서 오버피팅 발생 → 자율 rollout을 배치에 섞어 넣는 것이 필수.
  • 메모리 vs 성능(그림 4): FlowDAgger는 Assembly SR-peak memory 평면에서 좌상단(고성능·저메모리). SFT/LoRA는 우측(고메모리) 또는 하단(저성능).

Q추가 질문 & 답변

Q1. per-step 고정점 반전은 flow matching을 거꾸로 하는 거야? action을 noise로 바꾸는 거고, 수식이 있어?

맞다. FlowDAgger는 정책의 flow-matching 생성 과정을 거꾸로 돌려, 전문가 액션 a*를 그 (동결된) 정책이 만들어낼 noise 벡터 w*로 되돌린다(latent space에서 적응). 즉 action → noise 변환이 맞다.

  • 순방향 한 스텝(Eq. 2): xk+1 = xk + Δt·vθ(xk, tk, s)
  • 이를 뒤집으면 암시적 식 xk = xk+1 − Δt·vθ(xk, ·)가 되는데, xk가 양변에 있어 닫힌 형태로 못 푼다 → 고정점 반복으로 푼다(Eq. 3): xk(m+1) = xk+1 − Δt·vθ(xk(m), tk, s), 초기값 xk(0)=xk+1
  • 이를 xK=a*에서 시작해 backward로 x0=w*까지 복원.

리스트의 π0·Diffusion Policy가 쓰는 flow/diffusion 액션 헤드를 그대로 역방향으로 활용하는 셈.

Q2. diffusion(DDIM inversion)이랑 비슷한 거야?

비슷하지만 두 가지 수정이 필요하다. DDIM inversion은 diffusion 샘플의 결정론적 latent를 복원하는데, FlowDAgger는 이를 flow-matching 액션 헤드와 WAM으로 가져온다(§2). 핵심 차이(§4.1): 각 스텝을 암시적으로(고정점) 풀지, 한 번의 명시적 역스텝으로 근사하지 않는다. 명시적 역스텝은 속도 v를 xk 대신 xk+1에서 평가해 O(Δt) 오차가 생기는데, 이미지 diffusion(Δt~10−3)에선 무시할 만하지만 few-step 액션 헤드(K=10, Δt=0.1)에선 무시할 수 없다 — 그래서 고정점 반복이 "여기서 신뢰성 있게 만드는" 핵심이라고 논문은 말한다. 리스트의 WAM-TTT와 같은 '월드-액션 모델을 배포시 적응'하는 축의 다른 접근.

Q3. M=5는 무슨 뜻이야? 5번 반복하는 거야?

맞다. M은 inversion 스텝당 고정점 반복 횟수(Eq. 3의 m)다. M=5 = 각 스텝마다 5번 반복(기본값, §4.1). 반복이 기하급수적으로 수렴해 몇 번이면 충분한데, Table 6이 이를 뒷받침한다:

  • M=3 → 액션 MSE 0.00397
  • M=5 → 0.00168 (기본값)
  • M=10 → 0.00147 (개선 미미, 1.3배 느림)

즉 "per-step 수축이 M=5에서 사실상 수렴"하므로 기본값으로 채택했다. (참고: 여기 M과 별개로, flow 적분 스텝 수는 K=10.)
