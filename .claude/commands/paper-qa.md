논문 Q&A 저장: **$ARGUMENTS**

특정 논문에 대한 질문과 답을 정리해, 그 논문의 요약 페이지(`summaries/*.html`) 맨 아래
`추가 질문 & 답변` 섹션에 항목으로 추가한다. 대상 논문이 모호할 때만 확인한다.

## 배포 모드 (중요)
- **기본 = "추가 질문 확인 후 배포"**: 답변을 페이지에 삽입(로컬 저장)까지만 하고 **바로 커밋·push하지 않는다**.
  사용자에게 답을 보고한 뒤 **"이 논문에 대해 더 궁금한 점 있으세요? 없으면 배포하겠습니다"**라고 묻고 **멈춘다**.
  - 사용자가 추가 질문을 하면 → 같은 페이지에 이어서 항목을 더 삽입(여전히 배포 X), 다시 물어본다.
  - 사용자가 "없어 / 배포해 / deploy / 그만" 등으로 답하면 → 그때 **4-B(커밋·push)**를 실행한다.
- **즉시 배포 옵션**: `$ARGUMENTS`에 `!` 또는 `바로배포`/`즉시배포`/`지금배포`/`--push`/`--now` 중 하나가 있으면,
  추가 질문을 묻지 않고 삽입 직후 곧바로 커밋·push한다(이 키워드는 질문 텍스트에서 제외).

## 1. 대상 논문 파일 찾기
- `$ARGUMENTS`(또는 직전 대화 맥락)에서 논문을 식별한다: arXiv id 또는 제목/약칭.
- `summaries/` 에서 해당 파일을 찾는다(예: id로 `ls summaries/<id>_*.html`, 없으면 제목 키워드로 grep).
- 후보가 여러 개면 사용자에게 어느 것인지 짧게 확인한다. 하나면 그대로 진행.

## 2. Q&A 내용 준비 — 반드시 실제 논문에 기반
- **질문**: `$ARGUMENTS`에 질문이 있으면 그것을, 없으면 직전 대화에서 사용자가 이 논문에 대해 물은
  가장 최근 질문을 사용한다.
- **답변은 요약본이 아니라 실제 논문 전문에 근거해야 한다.** 답하기 전에:
  1. `papers/<...>fulltext*.md` 에 이 논문 전문이 있으면 그것을 Read 한다.
  2. 없으면 `https://arxiv.org/html/<id>` (전문 HTML)를 WebFetch 한다. 특정 절·표가 필요하면 해당 부분을 콕 집어 다시 fetch한다. 필요시 WebSearch로 보완.
  3. 요약 HTML(`summaries/*.html`)은 맥락 파악용 보조로만 쓰고, 그것만 보고 답하지 않는다.
- 이미 이번 대화에서 논문 전문을 읽고 답을 찾았다면 그 결론을 **핵심만 간결히** 정리한다(다시 읽을 필요 없음).
- 답은 한국어로, **논문의 구체적 근거(절·그림·표·수치)를 함께** 밝힌다. 논문에 명시가 없으면
  "논문에 명시 없음"이라고 쓰고 추정임을 표시한다. 수치·주장은 절대 지어내지 않는다.
- 길게 늘어놓지 말고 2~5문장 + 필요시 짧은 목록으로 정리한다.
- **관련 논문 링크**: 답변이 리스트의 다른 논문과 연결되면 상대경로 링크를 건다
  (예: `<a href="2602.10098_VLA-JEPA.html">VLA-JEPA</a>`). 대상 파일이 `summaries/`에 실제로 있는지 확인 후 링크.

## 3. 요약 페이지에 삽입 (접기/펼치기 형식)
대상 파일의 `<section class="qa">` 안을 수정한다. Q&A는 `<details>`로 만들어 **기본 접힘**, 질문 클릭 시 펼쳐진다.
모든 요약 페이지에는 접기용 CSS(`details.qa-item …`)가 이미 포함돼 있다(혹시 없으면 `.qa .empty{…}` 규칙 바로 아래에 추가). CSS는 그 외 건드리지 않는다.
- 기존 항목(`<details class="qa-item">`) 수를 세어 다음 번호(Q1, Q2, …)를 정한다.
- **첫 항목**: placeholder `<p class="empty">아직 추가 질문이 없습니다…</p>` 를 아래 블록으로 **교체**한다.
- **이후 항목**: 마지막 항목 뒤(섹션 `</section>` 직전)에 `<div class="divide"></div>` 와 함께 **덧붙인다**.

삽입 블록 형식:
```html
<details class="qa-item">
  <summary class="q">Q{n}. {질문}</summary>
  <p>{답변 핵심 정리}</p>
</details>
```
- 답변에 목록이 유용하면 `<ul><li>…</li></ul>` 사용 가능. HTML 특수문자는 이스케이프한다.

## 4. 삽입 후 처리 (배포 모드에 따라)
먼저 HTML 구조(섹션·태그 균형)를 점검한다. 그다음:

**A. 기본 모드 (즉시 배포 키워드가 없을 때):**
- git 커밋·push를 **하지 않는다.** 로컬 편집만 된 상태로 둔다.
- 사용자에게: 어떤 논문에 Q&A를 추가했는지 + 답변 요지를 짧게 보고하고,
  **"더 궁금한 점 있으세요? 없으면 배포할게요"**라고 물은 뒤 멈춘다.
- 사용자가 추가 질문을 하면 2~3 절차로 항목을 더 삽입하고 다시 묻는다(계속 미배포). 배포 승인이 오면 B로.

**B. 배포 (즉시 배포 옵션이거나, 기본 모드에서 사용자가 배포를 승인했을 때):**
  ```
  git add summaries/<파일>
  git commit -m "논문QA: <id/약칭> 질문 추가 — <질문 요약 10자>"
  git push
  ```
- 그동안 모은 여러 Q&A는 **한 번에 커밋·push**한다. 완료 후 커밋 해시와 몇 번째 Q&A인지 한 줄 보고한다.

## 참고
- 이 작업은 자동 스케줄 대상이 아니다(사용자의 질문·대화가 전제). 대화는 평소처럼 하고, 저장만 이 커맨드로 한다.
- 자연어로 "이 Q&A 요약페이지에 추가해줘"라고 해도 동일하게 처리한다.
